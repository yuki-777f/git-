# PDF 完成标准对应表

本文档把《手语数字人项目方案与分工.pdf》中进度计划的完成标准逐条落实到成员四交付物中。

## 第 1 周：环境配置与数据准备

- 所有成员可运行基础代码：三个模型均建立独立环境，并完成 compileall、官方 demo、评估脚本或最小推理测试。
- CSL_Daily 数据集可用：记录视频、标签、样本数量、路径和可读性；若模型主要依赖 PHOENIX14T，说明替代原因和统一评测对应关系。
- 熟悉项目代码结构：每个模型都写明 输入 -> 预处理 -> 模型 -> 输出 -> 指标 的数据流。
- 确定模型方案细节：SEDS 是评估工具，A Simple Baseline 是主生成路线，Sign-IDD 是高质量扩散生成对比路线。

## 第 2-4 周：模型开发与初步训练

- 搭建模型架构：克隆官方仓库后先做语法检查和最小运行，确保代码可运行、无语法错误。
- 完成 Stage1 预训练：完整预训练成本过高时，可使用官方预训练特征、预训练权重或小样本替代实验，但必须记录替代原因、使用资源和影响范围。
- 记录训练日志和指标：保存命令、日志、loss 曲线、验证指标、checkpoint 路径、硬件信息、运行时长和随机种子。

## 第 5 周：模型微调与优化

- 完成 Stage3 下游微调：能完整微调则优先完整微调；资源不足时完成官方 checkpoint 推理、小样本微调或可复现实验，并明确限制。
- 进行模型评估：SEDS 记录 Recall@K、MR、MedR；A Simple Baseline 记录 BLEU/BLEURT、回译分数、动作可视化和 SEDS 相似度；Sign-IDD 记录 BLEU/BLEURT、DTW、姿态距离、loss、可视化质量和 SEDS 相似度。
- 尝试超参数调优：至少记录学习率、batch size、epoch 或 steps、seed、输入长度、checkpoint 来源和调优前后指标变化。

## 第 6 周：结果对比与总结

- 统一评估指标：同一数据集、同一划分、同一硬件记录、同一模板汇总。
- 公平对比：区分官方预训练推理、小样本微调和完整训练，避免不同训练条件直接混比。
- 撰写实验报告：使用 06_progress_report.md 作为最终报告骨架。
- 确定最优方案：短期推荐 SEDS + A Simple Baseline，Sign-IDD 作为后续增强。
- 实时演示能力：给出 语音 -> 文本 -> 手语动作生成 -> 渲染或 UE 导入 的接入路径；若不能实时运行，至少提供离线生成视频或动作文件演示。
