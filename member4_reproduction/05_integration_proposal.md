# 与现有手语数字人系统的接入建议

## 当前系统现状
当前 elise 系统已经具备两条链路：语音转固定动画、手语视频转文字。语音链路目前通过 ASR 和大模型匹配 lexicon，再向 UE5 数字人发送 OSC 动画名；手语识别链路使用 Uni-Sign 输出文本。

## 成员四模型的接入角色
- SEDS：不直接接入实时链路，作为生成动作质量的自动评估工具。
- A Simple Baseline：主接入对象，用于把文本或语音转换为 SMPL-X 动作，再生成可视化 Avatar 或转换为 UE 可用动画。
- Sign-IDD：高质量扩散生成增强路线，先离线验证，再考虑接入。

## 推荐接入路径
1. 语音输入经过现有 ASR 得到文本。
2. 文本送入 A Simple Baseline 的 Text2Gloss 和动作生成模块。
3. 输出 SMPL-X 参数序列。
4. 离线阶段先用 Blender 或官方渲染脚本生成 Avatar 视频。
5. 工程阶段再研究 SMPL-X 到 UE5 骨骼、动画资产或控制曲线的转换。
6. 使用 SEDS 对文本和生成视频进行检索相似度评分，作为自动化质量检查。

## 演示优先级
第一阶段演示不强求实时，只需做到输入一句文本后生成手语动作视频。第二阶段再接入 elise 页面或 UE 数字人。第三阶段优化为接近实时。

## 风险与处理
- UE5 工程资产不完整：先做离线视频演示，不阻塞模型复现。
- SMPL-X 到 UE5 骨骼映射复杂：先输出标准 SMPL-X 和渲染视频，再单独做动画转换。
- 生成模型速度慢：先展示离线生成能力，实时能力作为后续优化。
- 中文手语数据不足：优先记录 CSL_Daily 可用性，必要时使用 PHOENIX14T 完成方法验证。
