## SEDS 环境

- Conda 环境：`seds-reproduction`
- Python：`3.10.20`
- NumPy：`1.25.1`
- Torch：`2.3.1+cu121`
- Torchvision：`0.18.1+cu121`

官方 README 要求 `torch==2.3.1+cu121` 和 `torchvision==0.18.1+cu121`。安装 `SEDS/requirements.txt` 时发现 `xformers==0.0.22.post7` 会要求 `torch==2.1.0`，导致 PyTorch 版本被替换并引发冲突。

处理方案：

- 生成 `member4_reproduction/environments/seds_requirements_no_xformers.txt`。
- 跳过 `xformers`。
- 重新固定 `numpy==1.25.1`、`Pillow==9.5.0`、`fsspec==2024.12.0`、`MarkupSafe==2.1.5`。
- 卸载冲突的 `xformers`。

验证命令：

```powershell
conda run -n seds-reproduction python -c "import numpy, torch, torchvision; print('SEDS', numpy.__version__, torch.__version__, torchvision.__version__)"
conda run -n seds-reproduction python -m compileall SEDS
```

验证结果：导入通过，`compileall` 通过。

## Spoken2Sign / A Simple Baseline Docker 环境

安装内容：

- Docker Desktop：已通过 `winget` 安装。
- WSL2：已启用。
- Ubuntu WSL 发行版：已安装。
- 官方基础镜像：`rzuo/pose:sing_ISLR_smplx`
- 本地镜像：`spoken2sign-reproduction:latest`

Docker Desktop 初次启动时出现：

```text
Docker Desktop is unable to start
```

排查后发现 Docker CLI 已安装、WSL2 可用、Ubuntu 可运行，但 Docker daemon 一直返回 503。通过重启 Docker 相关进程和 WSL 后恢复：

```powershell
Get-Process *docker* -ErrorAction SilentlyContinue | Stop-Process -Force
wsl --shutdown
Start-Process "C:\Program Files\Docker\Docker\Docker Desktop.exe"
```

之后 `docker info` 返回版本：`29.6.1`。

镜像拉取和构建：

```powershell
docker pull rzuo/pose:sing_ISLR_smplx
docker build -f member4_reproduction/environments/spoken2sign.Dockerfile -t spoken2sign-reproduction:latest .
```

镜像检查：

- `rzuo/pose:sing_ISLR_smplx`：约 `31GB`
- `spoken2sign-reproduction:latest`：约 `31GB`

首次 `compileall` 失败在 `SLRT/Spoken2Sign/vposer/._vposer_smpl.py`，原因是 macOS AppleDouble 残留文件含空字节，不是主代码错误。

修复和验证命令：

```powershell
docker run --rm -v "${PWD}/SLRT:/workspace/SLRT" spoken2sign-reproduction:latest bash -lc "cd /workspace/SLRT/Spoken2Sign && find . -name '._*' -delete && python -m compileall ."
```

验证结果：`Spoken2Sign` 容器内语法检查通过。
